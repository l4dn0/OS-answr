## 1. Что такое ОС?
Операционная система (ОС) - это комплекс программ, который управляет аппаратными ресурсами компьютера и предоставляет пользователям и прикладным программам удобный интерфейс для работы с аппаратурой.

## 2. Перечислить функции ОС как виртуальной машины
Согласно *Э. Танненбауму*  ОС как ВМ скрывает сложность аппаратного обеспечения и предоставляет удобные абстракции для работы:
* Предоставление интерфейса для работы с файловой системой вместо работы с секторами диска напрямую;
* Создание и управление процессами;
* Управление памятью - ОС создаёт абстракцию виртуальной памяти;
* Организация ввода-вывода - взаимодействие через единый интерфейс;
* Обеспечение пользовательского интерфейса (через CLI или GUI);

## 4. перечислить характеристики платформы Win32
1. 32-разрядная архитектура (использует 32-разрядные адреса и данные);
2. Вытесняющая многозадачность;
3. Многопоточность;
4. Каждый процесс имеет собственное виртуальное адресное пространство;
5. Объектная модель ресурсов (многие системные ресурсы представлены объектами, будь то процессы, потоки, файлы и т.д.)
6. Механизм доступа к объектам через дескрипторы
7. Наличие GUI (оконная модель и событийно-ориентированное программирование)

## 5. Что такое вытесняющая и кооперативная многозадачность?
### Кооперативная многозадачность
Процесс сам решает, когда передать управление другой задаче. Процесс выполняется до тех пор, пока добровольно не освободит процессор, ОС не может принудительно остановить процесс.
**Плюсы**:
+ Просто реализовать
+ Небольшие накладные расходы на переключение задач
**Минусы**:
- Один процесс может занять процессор на неопределённое время
- Ошибка в одной программе может привести к зависанию всей системы целиком
### Вытесняющая многозадачность
ОС сама управляет распределением процессорного времени. Каждому процессу выделяется квант времени, после окончания которого ОС может принудительно прервать процесс и передать процессор другому
**Плюсы**:
+ Ни один процесс не может монополизировать процессор
+ Более высокая надёжность
+ Более справедливое распределение ресурсов
**Минусы**:
- Более сложная реализация
- Дополнительные расходы на переключение контекста

## 6. Что такое изолированное и единое адресное пространство?
### Изолированное адресное пространство
Каждый процесс имеет собственную область виртуальной памяти, недоступной для других процессов. Один и тот же виртуальный адрес в разных процессах может соответствовать разным физическим областям памяти.
**Плюсы**:
- Высокая надёжность
- Защита процессов друг от друга
- Ошибка в одном процессе не повреждает данные других процессов
**Минусы**:
- Для обмена данными нужны специальные механизмы межпроцессорного взаимодействия (IPC)
- Дополнительные затраты на управление памятью.
Данный тип адресного пространства используется почти во всех современных ОС
### Единое адресное пространство
Все процессы используют общую область памяти. Любой процесс может обращаться к данным других процессов
**Плюсы**:
+ Простота обмена данными между процессами
+ Меньше накладных расходов на защиту памяти
**Минусы**:
- Отсутствие надёжной защиты
- Ошибка одной программы может повредить данные других программ или всей системы
Этот тип использовался в MS-DOS.
